# Documentation Map

## Authority order

1. Running source and automated tests.
2. `README.md`, `RELEASE_NOTES.md`, and `CHANGELOG.md` at the project root.
3. Current architecture and engineering handbooks.
4. Component specifications.
5. Historical reviews and prior-release reports.

## Current handbooks

- `ARCHITECTURE_HANDBOOK.md` — runtime boundaries and ownership.
- `ENGINEERING_HANDBOOK.md` — doctrine, change rules, and validation.
- `ADAPTER_CONTRACT.md` — required adapter interface.
- `SDK.md` — extension points used by adapter authors.
- `DISCOVERY_FRAMEWORK.md` — normalized discovery kinds.
- `CONVERSATION_SCHEMA.md` — operational conversation export contract.
- `KNOWLEDGE_OBJECT_SPEC.md` — additive next-generation artifact envelope.
- `PLUGIN_LIFECYCLE.md` — adapter installation, compatibility, and retirement.
- `COLLECTION_PLANNER.md` — future planner architecture; not implemented.
- `X_INTEGRATION.md` — current X/X Pro boundary and official-API adapter requirements.

## Validation

- `FINAL_VALIDATION.md` — authenticated live acceptance matrix.
- `PHASE_III_REGRESSION.md` — platform architecture regression gate.
- `TOPOLOGY_VALIDATION.md` — topology-specific safety and correctness.
- `ARCHITECTURAL_REGRESSION.md` — enduring architectural invariants.

## Historical context

`ENGINEERING_REVIEW.md`, `STABILITY_REPORT.md`, `TEST_REPORT.md`, `CHANGE_SUMMARY.md`, `LESSONS_LEARNED.md`, and the migration guide preserve the reasoning and evidence behind earlier releases. They are informative; they do not override current code or handbooks.
